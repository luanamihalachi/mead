MEAD (Medical Web Advisor) - User Guide

Tags: project, infoiasi, wade, web

This document explains how to use MEAD from an end-user perspective and includes screen captures.

1. Overview

MEAD is an educational web application that helps users explore countries and common diseases/conditions and obtain an explainable risk assessment for a selected country-disease pair. The application is designed for non-expert users, especially high-school students, and focuses on clarity and transparency.

2. Requirements and Access

  • A modern web browser (Chrome, Firefox, Edge).
  • Access to the MEAD frontend URL (local or deployed).

If running locally, ensure the backend services (geo-service, condition-service, risk-assessment-service) and Fuseki are started, then open the frontend in the browser.

3. User Interface Guide

3.1 Main Pages

  • Conditions view - selecting a disease/condition (asthma, obesity, depression).
  • Countries view - browsing or searching for a country.
  • Assessment view - showing the computed risk score, level, and explanation.

3.2 Typical Interaction Flow

  1. Select a country (by name or ISO2 code where applicable).
  2. Select a disease/condition.
  3. Request an assessment (the app computes or loads a cached result).
  4. Read the result: score, risk level, and explanation bullets.

4. Case Studies

This section demonstrates at least three practical scenarios using MEAD. Each case study contains the inputs, the steps a user performs, and the outputs displayed by the system.

Case Study 1: Country Snapshot Exploration (US)

Goal: View information about a selected country.

Inputs: Country = United States (ISO2: US)

  1. Open the Countries view.
  2. Select United States.
  3. Inspect the displayed indicators (population density, GDP, HDI, etc.).

Expected Output: A country snapshot with core indicators.

Case Study 2: Explainable Risk Assessment (US + Asthma)

Goal: Compute an explainable assessment for a country-disease pair.

Inputs: Country = US, Disease = asthma

  1. Select United States (US).
  2. Select the condition Asthma.
  3. Trigger the risk assessment.
  4. Read the result: level, score, and the explanation bullet list.

Expected Output: Risk level (low/medium/high), numeric score, and explanation bullets describing triggered rules.

Case Study 3: Exploring Disease Information (Asthma)

Goal: View detailed information about a disease.

Inputs: Disease = Asthma

  1. Open the Conditions (Diseases) view in the frontend.
  2. Select the condition Asthma from the list of supported diseases.
  3. Read the displayed description, causes, and listed risk factors.

Expected Output: A disease information view containing a short description of the condition, a list of causes, and relevant risk factors considered by the assessment logic.
